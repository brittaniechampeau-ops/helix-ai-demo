Execution
Each tool feeds the next. Discover is completed by Britt for the client — it assesses readiness across four dimensions, assigns RIVE scores, recommends a platform path, and sends the client a magic-link welcome email that drops them directly into their Vector workspace with their scores already loaded. Resolve locks in the client's structured vocabulary — audience segments, offer types, content categories. Intelligence extracts buyer signals, pain points, and market trends from the client's own documents. Vector loads that taxonomy and those signals as context before running its AI pipeline — so every piece of positioning it generates is grounded in the client's actual language and live market evidence, not generic templates. Engine takes Vector's output and generates campaign plans and DAM-ready content metadata, with taxonomy field values mapped directly to the client's marketing stack.
Core differentiator · The Enterprise Context Layer
The AI doesn't run until the client's context is loaded.
Most AI positioning tools generate from a blank slate — they produce generic outputs because they have no access to the client's actual language, buyer archetypes, or market evidence. DRIVE is different because of the context layer that loads into Vector before the AI pipeline runs.

Resolve taxonomy gives the AI the client's precise vocabulary: their specific audience segment names, offer type labels, and content category definitions — not generic stand-ins. Intelligence signals give the AI the client's own evidence: actual buyer quotes, confirmed pain language, competitive moves, and budget signals extracted from real documents. When both are loaded, the positioning that comes out of Vector's AI pipeline is built from the client's world — not assembled from training data averages.

Campaign DNA closes the loop. The positioning output maps back to the exact taxonomy field values the client's marketing stack already uses. HubSpot properties, Salesforce custom fields, Marketo tokens — the output is structured so it can be imported directly, without reformatting. The campaign doesn't just live in a strategy deck; it flows into every tool the client runs.

The tools · Step-by-step
D
Tool 01
Discover
Platform Readiness Assessment — Britt completes this for the client. Scores readiness across four dimensions, recommends a platform path, and sets up the client's workspace automatically.
How Discover works: Britt completes the assessment for the client — not the client themselves. Britt fills in the org context and answers the 16 readiness questions based on what she's learned about the client. When she saves the record, DRIVE creates the client's workspace and sends them a magic-link welcome email. The client opens the link, lands in Vector, and finds their Discover scores already loaded with a recommended platform path waiting for them.
01
Enter org context
Client name, industry, team size, revenue band, primary growth challenge, and primary contact email. This creates the workspace record and is the header of the readiness report. The primary contact email is used to send the welcome email.
02
Complete 4 sections × 4 questions each
The 16-question assessment covers Resolve readiness (taxonomy clarity), Intelligence readiness (signal availability), Vector readiness (strategy maturity), and Engine readiness (execution capacity). Each question is scored 0–2. Section scores run 0–8. Micro accounts route directly to Strategy and Execution panels; Resolve and Intelligence readiness are pre-scored based on tier defaults.
03
Review the RIVE readiness report
Scores across all four dimensions displayed as X/8, with a readiness level (Low / Medium / High) per dimension. The recommended platform path is derived from the score pattern. The report also shows a sequenced implementation roadmap and notes on which modules are unlocked.
04
Save to record — client workspace created
Saving writes the readiness scores and recommended modules to the practices table in Supabase (stored in the readiness jsonb column and modules_unlocked field). The client receives a magic-link welcome email. When they click through, Vector reads their discover record and surfaces the recommended track automatically.
05
Invite teammates
After the workspace is created, additional teammate emails can be added to team_emails on the practices record. Each teammate gets their own magic-link invite and shares the same org workspace — same taxonomy, same signals, same Vector state.

R
Tool 02
Resolve
Taxonomy Workbench — builds the client's structured vocabulary. Audience segments, offer types, content categories. This is the language layer that every other tool is built on.
Access: Subscription-gated. Clients with an active subscription can access Resolve. Admin emails bypass the gate. The taxonomy the client builds here flows into Vector's context layer and into Engine's asset tagging.
01
Review default taxonomy
Resolve loads with a pre-seeded taxonomy based on 18 practice archetypes. Review existing Audience segments, Offer types, and Content categories. Terms start as Pending until explicitly defined.
02
Define, edit, and add terms
Click any term to edit it. Use the + button to add new terms. Set each term to Defined (teal highlight) or leave as Pending (gray). Defined terms are what flow into Vector and Engine. Remove terms that don't fit the practice.
03
Add custom categories
The three default columns (Audience, Offer, Content) cover most practices. Add Category creates a new column for industry-specific segmentation — useful for practices that need separate dimensions like Buyer Role, Funnel Stage, or Geographic Market.
04
Taxonomy saves to org record
Resolve saves automatically to the org_taxonomy table in Supabase. All team members see the same taxonomy. Vector reads it during the AI pipeline step. Engine reads it for asset tagging. Campaign DNA exports it as CRM field value payloads.

Tool 03
Intelligence
Market Signals — uploads documents, extracts tagged buyer intelligence. Six signal types. Saves to the org's Supabase record so Vector's AI pipeline runs with real market evidence loaded.
Access: Subscription-gated. Admin emails bypass the gate. Signals extracted here are stored in org_intelligence and flow directly into Vector's AI pipeline context layer — they are not just stored, they actively inform every positioning output Vector generates.
01
Add Anthropic API key
Click the API dot in the topbar to open the key banner. Paste the key — stored in the browser's localStorage only, never on a server. This is the client's own key; API costs are theirs.
02
Upload documents
Drag in discovery call transcripts (.vtt, .txt), pitch decks (.pptx), market research, competitive intelligence, or win/loss reports (.xlsx, .csv). All parsed in the browser. No file size limit enforced at the UI layer — practical limit is Claude's context window.
03
Extract signals — six types
Claude reads each document and extracts tagged signals: BUYER_SIGNAL (intent and readiness cues), MARKET_SIGNAL (broad market dynamics), COMPETITIVE (competitor moves and positioning), TREND (emerging patterns), PAIN (confirmed friction the buyer articulated), BUDGET (spend signals and approval context). Each signal is quoted directly from the source document.
04
Signals save to org record → flow into Vector
Signals are saved to org_intelligence. A summarized signal digest is also generated and stored. When Vector's AI pipeline runs, it reads this digest as part of the context layer — so positioning agents are working from the client's real buyer language and market evidence, not generic assumptions.

Team access · Shared org workspace
Every DRIVE workspace is org-level, not individual. The primary contact (set during Discover) is the workspace owner. They can invite teammates who share the same workspace — same Resolve taxonomy, same Intelligence signals, same Vector state, same Engine assets. RLS in Supabase enforces that only the owning org can read or write their data: access is granted to the owner email and any email listed in team_emails on the practices record.
How teammates are invited
The workspace owner goes to the Team section in their workspace. They enter a teammate's email and click Invite. DRIVE adds that email to team_emails on the practices table and sends the teammate a magic-link invite. The teammate clicks through and lands in the same workspace — same data, same tools, same state.
Permissions model
Owner: can view team, add teammates, remove teammates, and manage subscription. Teammates: can view and edit all workspace content (taxonomy, signals, Vector state, Engine assets) but cannot add or remove other teammates, and cannot manage billing.
Supabase RLS enforcement
Row-Level Security policies check that the authenticated user's email matches either the email column (owner) or is listed in the team_emails jsonb array on the practices record. No cross-org data leakage is possible at the database layer.
What "shared" means in practice
If one teammate defines a new Resolve taxonomy term, every other teammate sees it immediately. If the account lead runs Vector's AI pipeline, the positioning output is visible to the whole team. Engine assets created by any team member live in the shared org record. There is one record, not one per person.
Data model · Supabase tables
All DRIVE data lives in a shared Supabase project. Tables are either platform-level (practices, leads, subscriptions) or org-level (org_* tables scoped per workspace). RLS policies enforce org isolation on all org tables.

Access gates · Subscriptions and admin bypass
Vector is accessible to any workspace with a valid magic-link session. Resolve, Intelligence, and Engine are subscription-gated — they check for an active subscription record on load. Admin emails (configured server-side) bypass all gates and can access every tool in every workspace.
Discover
Open to Britt (admin). No client-facing gate. Clients receive the output via welcome email and workspace pre-load, but don't interact with Discover directly.
Resolve
Subscription required. Checks subscriptions table for active status on load. Admin emails bypass. Gate is shown as an upgrade prompt, not an error.
Intelligence
Subscription required. Same gate as Resolve. Also requires an Anthropic API key (stored in localStorage) to run extraction. Admin bypass applies.
Vector
Accessible with a valid magic-link session. No subscription check. All workspace members can access. Discover scores and recommended path load automatically from the org record.
Engine
Subscription required. Same gate as Resolve and Intelligence. Reads Vector state automatically on unlock — no manual import step. Admin bypass applies.
Admin dashboard
admin.html shows all client submissions with RIVE score chips. Accessible to admin emails only. Allows Britt to view all workspace states, readiness reports, and subscription statuses in one view.
